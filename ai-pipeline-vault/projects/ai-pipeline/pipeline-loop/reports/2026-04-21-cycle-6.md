# Pipeline Cycle Report — cycle 6 (2026-04-21)

## Summary

Cycle 6 is a **net regression** driven primarily by BEC-58's "early iso tiebreaker" firing too aggressively. The fix worked mechanically — 7/10 clusters promoted (vs 3/10 cycle 5) — but **6 of those 7 promotions happened with `stability_score` of 0.0000-0.0833**, replacing older-but-better champions with brand-new untested cold-start models just because the champion wasn't iso-calibrated. The downstream backtest shows **every comparable profile/regime cell regressing** vs cycle 5: moderate/sideways Sharpe 3.86 → 0.80 (−79%), Info Ratio +0.93 → −2.44 (lost the first positive IR), aggressive/bull +0.31% → −4.59%. BEC-56 (per_cluster_overrides sync + substring match) worked as intended — no logs complaining about missing overrides; val_precision_up stayed flat or improved across all 10 clusters. BEC-57 (calibration-aware min_recall) is live (the `val_adjusted_min_recall` metric is logged) but still doesn't unblock candidates whose recall collapses to 0.000-0.03. BEC-59 degenerate-cluster handling didn't regenerate `clusters.parquet` (mtime still 03:52 from cycle 5, cluster membership unchanged; Miscellaneous 0.231 and ConsumerDef-Tech-Util 0.234 still show the degenerate silhouettes but no mitigation triggered this cycle). BEC-60 (T upper bound raised 2.5 → 4.0) is firing heavily: **8+ of 30 ensemble members hit T=4.0** exactly (up from 2.5 ceiling hits in cycle 5), confirming models remain pathologically overconfident, though isotonic still brings ECE to 0.0002-0.005 in every case.

## Suggested improvements (verdict=improve)

1. **Stage 3 — Add minimum-stability guard to BEC-58's early iso tiebreaker.** `src/evaluation/promote.py:104-121` currently promotes any iso-calibrated candidate over any iso=0 champion regardless of candidate quality. Cycle 6 promoted 5 candidates with stability_score in [0.0000, 0.0833] that directly degraded downstream portfolios. Options: (a) add `if cand_score < 0.15: skip early-exit` in the BEC-58 branch — falls through to normal cascading logic; (b) scale the floor by champion's own stability: `if cand_score < champ_score * 0.5: skip`; (c) require passed-plus-stability: `if cand_score < min(0.20, champ_stability): skip`. Expected impact: prevents cycle-6-style regression where 5 stability=0 models displace stale-but-stable champions; the calibrated candidates that DO have reasonable stability (Miscellaneous 0.32, CommunicationServices 0.49) still promote.

2. **Stage 3 — BEC-57 recall scaling doesn't fix the collapse; need top-K or percentile recall.** Even with `adjusted_min_recall` logged, `val_recall_at_primary` is 0.000-0.029 for every failed_recall candidate. The root issue: after isotonic calibration, fewer than 1% of predictions exceed any threshold ≥0.50, so recall-at-threshold is near-zero everywhere. Options: (a) replace `min_recall_at_primary_threshold` with `min_recall_at_top_percentile` (e.g. recall over the top-10% of predictions, which is always non-zero); (b) use `recall_at_calibrated_median + 0.5*sigma` — threshold adapts to per-cluster distribution; (c) remove the recall filter for iso-calibrated candidates (isotonic already guarantees probability-recall monotonicity). Expected impact: 3-5 more candidates pass Stage-3 filters on their own merits, reducing reliance on BEC-58's early-exit.

3. **Stage 1 — Trigger BEC-59 degenerate-cluster handling on cached clusters.** BEC-59 merged but clusters.parquet is cached (mtime 04-21 03:52); the merge/split path runs only when clustering itself regenerates. Miscellaneous (sil=0.231) and ConsumerDef-Tech-Util (sil=0.234) remain degenerate two cycles running. Options: (a) at `make pipeline` entry, check `silhouette_mean_cluster < 0.30` for any cluster and force rebuild; (b) run `src.features.clustering` with `--post-process` flag that applies BEC-59 logic to existing parquet without full recluster; (c) in Stage 2, skip training for clusters with silhouette < 0.25 and fall back to a global model prediction. Expected impact: 2 degenerate clusters (153/504 symbols) either get subdivided for cleaner training signals or flagged to use global model fallback.

4. **Stage 2 — Add overfit-gap regularization to the loss directly (not just Optuna objective).** BEC-53 overrides (hidden=[64], dropout≥0.40, mask=0.25) are applied in cycle 6, but cycle-6 still shows val_precision 0.33-0.52 and val_acc 0.44-0.54, implying the models aren't learning robust patterns — they're just being capped at small capacity. BEC-60's T=4.0 bound hits in 8+ runs, confirming train-side overconfidence persists. Options: (a) add a confidence-penalty to training loss: `loss += 0.1 * (max_prob - 0.85).clamp(min=0).mean()` — penalizes train predictions >0.85; (b) add an explicit label-smoothing floor of 0.10 (currently tuned to 0.02-0.12 by Optuna); (c) add weight-decay floor of 0.05 in overrides. Expected impact: fewer pathologically overconfident models, lower T scaling needed, better-calibrated probabilities entering Stage 3.

5. **Stage 6 — Add regression guard: if cycle-N backtest Sharpe drops >30% vs cycle-(N-1) on moderate/sideways, flag and skip champion replacement.** BEC-58 had no downstream safety check; we merged, ran full cycle, and only saw the −79% Sharpe regression after 3h20m. Options: (a) add `make backtest-diff` target that reads previous backtest report and warns if ANY comparable cell regresses >30%; (b) add a promotion circuit-breaker: if aggregate prob_up mean changes >0.10 vs previous cycle, block promotion until manual review; (c) track a "promotion health score" in MLflow (avg stability_score of newly promoted models) and alert when it drops <0.15. Expected impact: catches future BEC-58-style regressions in 5 minutes instead of 3 hours; avoids cascading into portfolio/backtest.
